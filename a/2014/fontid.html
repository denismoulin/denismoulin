Font Identity

Information Design, Product Design, Ux, Ui, Prototyping

Along with the platform and the sync app, I worked with Fontyou on a third project called the Font ID. It was a really challenging subject, as we engaged with information structure, exploring infinite cases and uses.

It’s a live object, meant to be both complementary and central to both the Fontyou platform and synchronising app. A Font ID is like the identity card for a font object. As the project evolves from a single post to a font in use, the Font ID is supposed to evolve too.

Here’s a quick look at the information structure. For example, the ID for two phases, a post and a font family for sale.

Each Font ID is different because each post and the interactions that rise from it are different. Also, each action would be different depending on its current state and the rights you have to it (whether you’re a user, the owner, a co-creator…).

The Font ID is central, available both on the synchronizer and on the platform.
